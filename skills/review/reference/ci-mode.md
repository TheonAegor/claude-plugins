# CI Mode Instructions

Note: The PR branch is already checked out in the current working directory.

## Step 1: Get the Diff

Run `gh pr diff $1` to get the PR diff. Also run `gh pr view $1 --json title,body` to get PR metadata.

## Step 2: Review the Changes

1. Review the diff following [Code Review Guide](${CLAUDE_PLUGIN_ROOT}/skills/review/reference/code-review.md)
2. Follow the CLAUDE.md guidelines for code review tasks

## Step 3: Post Review as GitHub Comments

**CRITICAL: You MUST post your review as GitHub comments.**

### For Automatic PR Reviews

- Use `gh pr comment` to post your overall review summary
- Use `mcp__github_inline_comment__create_inline_comment` for specific code issues
- Follow the priority matrix from code-review.md (Critical/High issues first)
- Include file:line references for all findings

### For @claude Mentions in Comments

- Always respond using `gh pr comment`
- Provide actionable recommendations
- Reference specific code locations when relevant

### Comment Format

- Start with "## Claude Code Review" or "## Claude Analysis"
- Use markdown formatting for readability
- End with "*Generated by Claude Code*"
